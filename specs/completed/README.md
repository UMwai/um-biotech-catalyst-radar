# Completed Features - Archive

Specifications for features that have been **shipped to production**.

## Why Archive Specs?

1. **Historical reference** - Understand original requirements vs final implementation
2. **Onboarding** - New contributors can see how features were designed
3. **Iteration** - When enhancing features, review original spec first
4. **Compliance** - Document what was promised vs delivered
